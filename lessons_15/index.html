חלק ב׳ — try / except

תחביר — לנסות ולתפוס

מבנה בסיסי
try:
    # קוד שעלול לגרום שגיאה
    num = int(input("הכנס מספר: "))
    print(100 / num)

except ValueError:
    print("זה לא מספר! נסה שוב.")

except ZeroDivisionError:
    print("אי אפשר לחלק באפס!")
איך זה עובד
Python מנסה להריץ את הקוד שב-try. אם משהו נכשל — היא קופצת ל-except המתאים ומריצה אותו. הקוד לא מתרסק — הוא ממשיך בצורה נשלטת.

הקוד מאחורי המחשבון
def safe_divide(a, b):
    try:
        result = float(a) / float(b)
        return f"תוצאה: {result:.2f}"
    except ValueError:
        return "שגיאה: הכנס מספרים בלבד"
    except ZeroDivisionError:
        return "שגיאה: אי אפשר לחלק באפס"
except Exception — תופס הכל
אם לא יודעים איזו שגיאה תקרה, אפשר לכתוב except Exception as e: — תופס כל שגיאה, ו-e מכיל את הפרטים. שימושי אבל לא מדויק — עדיף לתפוס שגיאות ספציפיות.

לחץ על כל שאלה לתשובה.

1. מה קורה אם הקוד שב-try לא גורם שגיאה?
לחץ לתשובה
הקוד שב-except פשוט לא רץ. Python מריצה את ה-try, הכל עובד בסדר, וממשיכה לקוד שאחרי כל ה-try/except. except רץ רק כשיש שגיאה.
2. מה ההבדל בין except ValueError ל-except Exception?
לחץ לתשובה
except ValueError תופס רק שגיאות מסוג ValueError. except Exception תופס כל שגיאה אפשרית. עדיף להיות ספציפי — כך אפשר להגיב בצורה שונה לכל סוג בעיה.
3. למה כדאי להשתמש ב-try/except גם בתוכניות פשוטות?
לחץ לתשובה
כי משתמשים לא תמיד עושים מה שמצפים מהם. הם מקלידים טקסט במקום מספר, לוחצים Enter בלי להזין כלום, מכניסים ערכים קיצוניים. try/except הופך תוכנה שבירה לתוכנה אמינה.
