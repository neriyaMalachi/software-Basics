חלק ג׳ — קלט של מספרים

input() תמיד מחזיר טקסט!

זה חשוב מאוד: גם אם המשתמש הקליד את המספר 5, input() יחזיר את ה-string "5" — לא את המספר 5. כדי לחשב עם זה, צריך להמיר.

קלט של מספר — int() ממיר string למספר
age_text = input("כמה אתה בן? ")
age = int(age_text)        # ממיר string למספר שלם
next_year = age + 1

print(f"בשנה הבאה תהיה בן {next_year}")

# או בשורה אחת:
age = int(input("כמה אתה בן? "))
שגיאה נפוצה
אם המשתמש יקליד "שלום" במקום מספר, int() יזרוק שגיאה. בשיעור 15 נלמד איך לטפל בזה נכון עם try/except.

תרגיל בדוק את עצמך

לחץ על כל שאלה לתשובה.

1. למה input() תמיד מחזיר string גם אם הקלדנו מספר?
לחץ לתשובה
כי Python לא יודע מה המשתמש הולך להקליד לפני שהוא מקליד. אז היא תמיד שומרת את הקלט כטקסט (string). אנחנו צריכים להגיד לה בעצמנו: "זה מספר" — ולכן משתמשים ב-int() לאחר מכן.
2. מה ההבדל בין print() ל-input()?
לחץ לתשובה
print() שולח מידע החוצה — מציג למשתמש. input() מקבל מידע פנימה — מקשיב למשתמש. print = פלט, input = קלט.
3. מה יקרה אם נכתוב: age = input("גיל: ") ואז print(age + 1)?
לחץ לתשובה
שגיאה! כי age הוא string (טקסט), ולא ניתן לחבר string עם מספר (1). הפתרון: age = int(input("גיל: ")) — כך age יהיה מספר אמיתי.
