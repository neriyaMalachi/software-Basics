חלק א׳ — למה קבצים?

מידע שנשאר אחרי שסוגרים

עד עכשיו כל המידע שיצרנו "נעלם" ברגע שסגרנו את Colab. קבצים מאפשרים לשמור מידע לצמיתות — ולקרוא אותו שוב בהפעלה הבאה.

"r"
Read
קריאה בלבד. הקובץ חייב להתקיים.
"w"
Write
כתיבה. יוצר קובץ חדש או מוחק קיים.
"a"
Append
הוספה לסוף קובץ קיים, לא מוחק.
כתיבה וקריאה — הבסיס
# כתיבה — יוצר קובץ חדש
with open("notes.txt", "w", encoding="utf-8") as f:
    f.write("שלום עולם!\n")
    f.write("זו שורה שנייה.\n")

# קריאה — מחזיר את כל התוכן
with open("notes.txt", "r", encoding="utf-8") as f:
    content = f.read()
    print(content)
with — הדרך הנכונה לפתוח קבצים
with open(...) as f: פותח את הקובץ ואוטומטית סוגר אותו בסוף הבלוק — גם אם קרתה שגיאה. תמיד השתמש ב-with, לא ב-open רגיל.
חלק ב׳ — הוספה לקובץ

append — לא למחוק, רק להוסיף

יומן אישי — כל הרצה מוסיפה שורה
from datetime import date

today = str(date.today())
entry = input("מה קרה היום? ")

with open("diary.txt", "a", encoding="utf-8") as f:
    f.write(f"[{today}] {entry}\n")

print("נשמר ✓")
חלק ג׳ — נסה בעצמך

יומן אינטראקטיבי (בדפדפן)

סימולציה של יומן — נשמר ב-localStorage
לחץ "קרא יומן" לראות רשומות קודמות
encoding="utf-8"
תמיד הוסף encoding="utf-8" כשעובדים עם עברית בקבצים. בלי זה Python עלולה לא לקרוא את האותיות נכון.
תרגילבדוק את עצמך

לחץ לתשובה.

1. מה ההבדל בין מצב "w" לבין "a"?
לחץ לתשובה
"w" (write) מוחק את כל תוכן הקובץ הקיים ומתחיל מחדש — כמו גיליון ריק. "a" (append) מוסיף לסוף מה שכבר קיים — כמו יומן שאתה ממשיך לכתוב בו. אם אתה כותב יומן, תמיד השתמש ב-"a".
2. למה משתמשים ב-with ולא בפשוט open()?
לחץ לתשובה
with סוגר את הקובץ אוטומטית בסוף הבלוק — גם אם קרתה שגיאה באמצע. קובץ פתוח ששוכחים לסגור עלול לגרום לבעיות. with מונע זאת ומקצר את הקוד.
3. איך קוראים את הקובץ שורה שורה ולא הכל בבת אחת?
לחץ לתשובה
במקום f.read() משתמשים ב-f.readlines() שמחזיר רשימה של שורות, או עוברים ישירות עם for: for line in f: print(line) — Python עוברת שורה שורה אוטומטית.
